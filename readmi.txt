Для первого коммита (один раз для проекта):
-------------------------------------------
git init                         # Инициализируем пустой Git-репозиторий в папке (только 1 раз)
git add .                        # Добавляем все файлы в индекс (подготовка к коммиту)
git commit -m "Описание"         # Создаём первый коммит
git remote add origin https://github.com/Sapronov-Nikolay/user-database.git # Привязываем удалённый репозиторий на GitHub (только 1 раз)
git branch -M main                # Переименовываем текущую ветку в main (если Git создал master)
git push -u origin main           # Отправляем коммит на GitHub и запоминаем связь (флаг -u)

Для следующих изменений (каждый раз, когда что-то поменял):
-----------------------------------------------------------
git add .                        # или git add имя_файла (например, git add utils/plural.js)
git commit -m "Описание правок"
git push                         # Теперь можно просто отправлять на GitHub

ЕСЛИ СЛУЧИЛОСЬ РАЗЪЕДИНЕНИЕ ИЛИ РАЗОМКНУТО СОЕДИНЕНИЕ С GitHub ПО ЛЮБОЙ ПРИЧИНЕ
Вариант 1: Клонирование (проще всего)
Теперь можно сразу продолжать работать: git add, git commit, git push – всё уже готово.
Этот способ хорош ещё тем, что внутри папки сразу лежит актуальная версия с GitHub.
-------------------------------------
git clone https://github.com/Sapronov-Nikolay/user-database.git
cd user-database

Вариант 2: Восстановление связи у существующей папки
----------------------------------------------------
1. Открой терминал в этой папке.
2. Проверь, помнит ли Git старый remote:
    git remote -v
    Если покажет origin https://... – связь есть ИНАЧЕ ПРОВЕСТИ:
    git remote add origin https://github.com/Sapronov-Nikolay/user-database.git
3. Убедись, что ветка называется main:
    git branch -M main
4. Теперь можно выполнить git pull origin main, чтобы скачать последние изменения с GitHub (если они есть),
  а дальше работать как обычно: git add, git commit, git push.

ВЫВОД!!!
Новый компьютер / чистая установка → git clone (получить свежую версию и автоматическую связь).
Перенёс папку с рабочего ПК (в ней есть .git) → git remote add origin или git remote set-url origin, чтобы восстановить ссылку.
    git remote set-url origin Эта команда предназначена для изменения URL-адреса уже существующего удалённого репозитория,
    и она требует указать новый URL.
    если ты хочешь изменить URL (например, если репозиторий переехал или ты пересоздал его),
    тогда используй set-url с указанием нового адреса:
Таким образом:
  git remote -v → показать текущий URL.
  git remote set-url origin <новый URL> → изменить URL.
  Например: git remote set-url origin https://github.com/Sapronov-Nikolay/новое_имя.git
